# Google Form 5 題回饋表 — 完整設計與設置指南

> 用途：W1 朋友圈試用啟動後，收 30 個朋友的回饋
> 設計目標：以最少題目換取最大行動資訊（NPS + 痛點 + 變現訊號）
> 撰寫日期：2026-05-15

---

## 設計理念

5 題太多會降填寫率（業界 5 題以下完成率 ~70%、超過 7 題掉到 30%），所以每題都必須**換到具體決策**。

這份 5 題各自對應到您要做的決定：

| 題號 | 解答的決策 |
|------|----------|
| Q1（NPS）| 整體值不值得繼續做（< 30 就停損） |
| Q2（為什麼）| 最大賣點 vs 最大痛點是什麼（決定下一輪推什麼） |
| Q3（最有用功能）| 行銷時主打哪個賣點（決定 SEO 文章主題） |
| Q4（最想改的地方）| 下個月優先修哪個 bug / 加哪個功能 |
| Q5（願付金額）| 驗證商業模式（決定要不要走付費路線） |

## 5 題完整內容

### Q1：整體推薦度（量化指標）

**題目**：
```
從 0-10 分，您會把人脈手札推薦給朋友嗎？
```

**題目類型**：`Linear scale`（線性刻度）

**設定**：
- 從 `0`（完全不會）到 `10`（一定會）
- **設為必填** ⭐
- 0 的標籤：`完全不會`
- 10 的標籤：`一定會`

**這題為什麼這樣設計**：
這是 NPS（Net Promoter Score）的標準問法，全世界最被驗證過的滿意度指標。0-6 分是「貶損者」，7-8 是「中立」，9-10 是「推薦者」。

---

### Q2：為什麼給這個分數（開放式 — 最關鍵的一題）

**題目**：
```
為什麼給這個分數？(可以講優點也可以講缺點，越具體越好)
```

**題目類型**：`Paragraph`（段落文字）

**設定**：
- 設為必填 ⭐
- 提示文字 placeholder：`例：很喜歡命理分析的角度，但加人脈的流程太多步`

**這題為什麼這樣設計**：
NPS 分數沒有 Q2 是廢的。**真正有用的是這題**。10 分給「為什麼」的人會告訴您去哪打廣告（用他們的話），0 分給「為什麼」會告訴您要修什麼。

---

### Q3：目前最有用的功能（決定行銷主打）

**題目**：
```
您覺得人脈手札目前最有用的功能是？（單選）
```

**題目類型**：`Multiple choice`（單選題）

**選項**（請完整複製到 Google Form）：
- 八字命盤分析
- 星座/西方占星分析
- 數字易經
- 合作分析（雙向視角）
- 名片 OCR（拍照辨識）
- 純粹當人脈管理工具
- 還沒找到特別有用的地方

**設定**：
- 設為必填 ⭐
- ✅ 勾選「Other...」讓回答者能補充選項

**這題為什麼這樣設計**：
您現在不知道 12 個朋友裡有多少人是衝著「八字」來、多少是衝著「合作分析」來。這題給您**行銷時要主打哪個賣點**的數據答案。例：若 60% 選「合作分析」，下個月 SEO 文章主題就鎖定合作分析，不要花時間寫「12 星座運勢」。

---

### Q4：最希望改進的地方（決定下個月開發重點）

**題目**：
```
有什麼您覺得卡卡的、不順手、或希望能加的功能嗎？
```

**題目類型**：`Paragraph`（段落文字）

**設定**：
- 設為**非必填**（讓「沒意見」的人能跳過）
- 提示文字 placeholder：`例：希望可以匯入手機通訊錄；命理結果想要可以分享；切換東西方時會卡一下`

**這題為什麼這樣設計**：
**比 Q2 更精準的痛點訊號**。Q2 是「整體感受為什麼這樣」，Q4 是「具體想看到改變」。每個月把收到的 Q4 答案分類，最多人提到的那個就是下個月做的事。

---

### Q5：付費意願（變現訊號）

**題目**：
```
如果未來推出付費方案，您願意付錢的選項？（可複選）
```

**題目類型**：`Checkboxes`（複選題）

**選項**（完整複製）：
- NT$890 一次買斷終身使用（無限人脈 + 深入識人 AI 無限次）
- NT$99/月訂閱（含雲端同步，可跨手機/電腦）
- NT$30/次的深入識人 AI 分析（單次購買）
- 我覺得目前免費功能就夠了
- 我願意付錢，但我覺得價格應該是 ___（請在「其他」填寫）

**設定**：
- 設為必填 ⭐
- ✅ 勾選「Other...」

**這題為什麼這樣設計**：
這題等於**讓使用者投票您的三層收費結構（Lifetime / Cloud Sync / Per-event）哪個有市場**。如果 50% 選 NT$890 買斷、10% 選 NT$99 訂閱，您就知道訂閱模式不該優先做。

## 結尾選填欄位

加在 Q5 之後（**非必填**）：

### 留下聯絡方式
**題目**：
```
（選填）留下 Line ID 或 Email，方便我後續若有問題能簡短訪談 5 分鐘。
我不會打擾您、不發垃圾訊息。
```

**題目類型**：`Short answer`（簡答）

**設定**：非必填

### 訪談意願（選填）
**題目**：
```
（選填）我願意接受 15 分鐘電話訪談，幫助這個工具變更好。
```

**題目類型**：`Multiple choice`（單選）

**選項**：
- 願意，我留了聯絡方式
- 暫時不方便

**設定**：非必填

## Google Form 操作步驟（非技術人版）

### 第一次建立

1. 打開 https://forms.google.com（用您的 Google 帳號登入）
2. 點 **「Blank」**（空白表單）或大大的 `+` 號
3. 把表單標題改為 **「人脈手札 — 早期使用者回饋」**
4. 把表單說明改成上面提供的版本
5. 依照下面對應每題操作

### 加每一題的操作

#### Q1 Linear scale

1. 點右側 `+` 加新題目
2. 題目類型下拉選 **「Linear scale」**
3. 題目欄輸入 Q1 題目
4. 數字範圍設 0 → 10
5. 左邊標籤填 `完全不會`、右邊標籤填 `一定會`
6. 右下角 **必填開關打開** ⭐

#### Q2 Paragraph

1. 加新題目，類型選 **「Paragraph」**
2. 題目欄輸入 Q2 題目
3. 點題目下方提示「Description」加上 placeholder 引導文
4. 必填開關打開 ⭐

#### Q3 Multiple choice

1. 加新題目，類型選 **「Multiple choice」**
2. 題目欄輸入 Q3 題目
3. 一個一個 Enter 換行加 7 個選項
4. 點選項下方 **「Add 'Other'」**
5. 必填開關打開 ⭐

#### Q4 Paragraph

1. 加新題目，類型選 **「Paragraph」**
2. 題目欄輸入 Q4 題目
3. 加上 placeholder 引導文
4. 必填**不要打開**（讓沒意見的人能跳過）

#### Q5 Checkboxes

1. 加新題目，類型選 **「Checkboxes」**
2. 題目欄輸入 Q5 題目
3. 加 5 個選項
4. 點 **「Add 'Other'」**
5. 必填開關打開 ⭐

#### 結尾 Short answer + Multiple choice

1. 加 Short answer 題：留聯絡方式（**非必填**）
2. 加 Multiple choice 題：訪談意願（**非必填**）

### 設定感謝頁

1. 右上角點齒輪圖示 **「Settings」**
2. 切到 **「Presentation」** 分頁
3. 找到 **「Confirmation message」**
4. 貼上上方感謝訊息內容
5. 儲存

### 發佈 / 拿連結

1. 右上角點 **「Send」** 按鈕
2. 切到 🔗 圖示分頁（連結）
3. 勾選 **「Shorten URL」** 得到短網址
4. 把短網址複製下來

### 看收到的回應

1. 表單頁面點上方分頁 **「Responses」**
2. 預設會顯示彙整圖表（NPS 分布、Q3 選項統計）
3. 點 **試算表圖示** 連到 Google Sheet 看每筆細節

## 收到回饋後怎麼讀（分析建議）

### 每週日花 20 分鐘做這四件事

1. **看 NPS 分布**
   - 9-10 分人數 ÷ 0-6 分人數 = 推薦比
   - 推薦比 > 1：產品方向對，可以加速行銷
   - 推薦比 < 0.5：產品本身有問題，先修再推

2. **讀 Q2（為什麼）找 pattern**
   - 把高分（9-10）的「為什麼」抄一頁：這是您的**行銷文案**
   - 把低分（0-6）的「為什麼」抄一頁：這是您的**待修清單**

3. **看 Q3 統計**
   - 最多人選的功能 → 下週 SEO 文章主題
   - 沒人選的功能 → 下個月評估是否該砍掉（功能太多反而稀釋價值）

4. **整理 Q4 改進建議**
   - 把所有 Q4 回答貼到一個試算表
   - 按提到次數排序
   - 提到 3 次以上的進「馬上做」清單

### 一個月後（30 份）盤點

收到 30 份後做一次大盤點：
- 算 NPS 分數（推薦者% - 貶損者%）
- 看 Q5 統計：哪個價位方案最多人勾？
- 寫一篇「30 個使用者教我的事」內部 retrospective

## 變奏版（之後可能用到）

### 第 3 個月版本（針對 100+ 註冊）

當註冊數破百後，可改成這 5 題：

1. 您是從哪裡知道人脈手札的？（單選，含 KOL 名單）
2. 您最常用人脈手札來做什麼？（多選）
3. 1-10 分推薦度
4. 為什麼？
5. （付費方案上線後）您買的方案是什麼 / 沒買的原因？

### 付費使用者專屬版（M6 之後）

對付費客戶另發一份：

1. 是什麼讓您決定付費？
2. 付費後最常用的功能是？
3. 期待我們做什麼新功能？
4. 您會推薦同事/朋友也付費嗎？為什麼？
5. 1-10 分滿意度 + 留 Line 給訪談
