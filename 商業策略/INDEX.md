# 商業策略 — 文件導讀

> 進來先看這份。每個檔案的用途、何時看、看完該做什麼。

---

## 🎯 從這裡開始

### 想做行銷變現 → 看這份
**`變現執行藍圖_12個月.md`**

把所有策略接成可執行時程。看完就知道「下週做什麼、下個月做什麼、第 6 個月做什麼」。

### 競品價格與修正版策略 → 看這份
**`競品調查與商業路線修正_2026-05-15.md`**

調查名片王、先知命局的真實收費機制，據此修正人脈手札的三層定價（Lifetime / Cloud Sync 訂閱 / Per-event）與第 4-8 月執行步驟。**比原藍圖更貼近真實市場**。

### 每週要做什麼 → 看這份
**`每週執行清單_2026-W20開始.md`**

把 12 個月藍圖拆成 48 週的具體任務（🛠 Dev / 📣 Mkt / 📊 Data / 💰 Mon / 📝 Ops 五類）。每週日晚上看下週清單。每個階段都有 Stage Gate（通不過不能進下一階段）。

## 📊 商業模式分析

### 文字版（看完搞懂為什麼這樣定價）
**`商業評估.md`**

- 成本結構（每使用者每月 NT$5）
- 競品定價對標（台灣 sweet spot：月 99–149）
- 收費模型 ABCD 比較
- 達標時程估算

### 互動版（可拖滑桿模擬）
**`商業評估-人脈手札.html`** — 訂閱制模式試算（損益平衡 / 12 月預估）
**`商業評估-用量門檻模式.html`** — 用量門檻制（免費 N 人 → 付費）對比訂閱制

> 💡 建議：先看 `.md` 理解邏輯，再用 `.html` 調參數試不同情境

## 📁 用對檔案的決策樹

```
您現在的問題是？

├─ 「我下週/下個月該做什麼？」
│   → 變現執行藍圖_12個月.md
│
├─ 「該收多少錢？什麼模式？」
│   → 競品調查與商業路線修正_2026-05-15.md（最新建議）
│   → 商業評估.md（讀邏輯）
│   → 商業評估-人脈手札.html（試算）
│
├─ 「免費 vs 用量收費哪個比較好？」
│   → 商業評估-用量門檻模式.html
│
├─ 「怎麼找用戶？哪個渠道最划算？」
│   → 行銷策略-增長計劃.html
│
├─ 「下一步該開發什麼功能？」
│   → 產品路線圖.md
│
├─ 「有人提的功能要做嗎？」
│   → 需求優先序_2026-05-14.md（先對照原則）
│
└─ 「介面要改哪裡？」
    → UX設計提案.md
```

## ✏️ 維護建議

- **每季更新一次** `變現執行藍圖_12個月.md` 的進度欄
- **數字有重大變化時** 重算 `商業評估.md`（例如 API 漲價、新通路）
- **每月跑完月度數據** 把實際 vs 目標填到藍圖最後的儀表板
- **新功能想法** 全部先進 `待辦清單.md`（根目錄），每月初評估是否搬到 `需求優先序_YYYY-MM-DD.md`
