# 人脈手札 — 商業評估與收費方案

> 本文評估「人脈手札」的成本結構、收費模型、單位經濟、規模情境模擬。
> 跟 `產品路線圖.md` 對齊：Phase 2 開始驗證收費假設、Phase 3 正式上付費。
> 數字都是估算，跑出第一個月真實數據後要重評。

---

## 1. 成本結構（先算清楚才能定價）

### 1.1 固定成本（不管使用者多少都要付）

| 項目 | 月成本（NT$） | 備註 |
|---|---|---|
| Cloudflare Workers + Pages | **0** | 免費額度 100K req/day，足夠到 ~1500 active users |
| Cloudflare KV | **0** | 免費額度 100K reads + 1K writes/day |
| GitHub repo | **0** | Public repo 免費 |
| Domain（建議 Phase 2 買） | **25** | 例如 `.app` / `.tw` 約 NT$300/年 |
| **小計** | **~25** | 早期幾乎可忽略 |

### 1.2 變動成本（隨使用者數成長）

每個 active 使用者每月預估成本（保守估算）：

| 項目 | 用量 | 單價 | 月成本 (NT$) |
|---|---|---|---|
| Anthropic Haiku 4.5（免費版深入識人） | 5 次 × ~5K tokens | $0.03/次 | ~4.5 |
| Anthropic Sonnet 4.5（OCR 走 proxy 後） | 2 次 × ~2K tokens | $0.01/次 | ~0.6 |
| Cloudflare 流量 | 小（單檔 HTML） | — | 0 |
| **每 active user 月成本** | | | **~5 NT$** |

> ⚠️ 這假設使用者每月只跑 5 次深入識人。重度使用者（每天 1 次）會跑到 NT$30+/月。
> rate limit 設 3 次/IP/日 = 月上限 90 次 = **NT$80+/月/IP**（最壞情況）。

### 1.3 您自己的時間成本

| 工作 | 每月小時 | 時薪假設 NT$500（兼職機會成本） |
|---|---|---|
| 看 analytics + 回饋 | 2 | 1,000 |
| 寫 code（含 Claude 協作） | 6 | 3,000 |
| 客服 / 回 email | 2 | 1,000 |
| **小計** | **10 hrs/month** | **~5,000** |

> 不算進金錢成本（您不付自己錢），但**規劃時要當作真實資源**。
> 收入要能 cover 機會成本（即使 NT$5000/月）才不會疲乏放棄。

選擇相近定位的個人 CRM / 人脈管理工具：

| 工具 | 模式 | 定價（美金） | 換算 NT$ | 備註 |

| **Notion**（拿來做 CRM） | Freemium | Free / $10/mo | 月 300 | 通用工具 |
| **iOS 通訊錄 + 第三方 App** | — | NT$30-150 一次 | — | App Store 一次性 |

### 2.1 台灣市場價格敏感度修正

台灣使用者願付價約為美國的 **1/3 ~ 1/2**：
- 月訂閱 sweet spot：**NT$59-149**
- 年訂閱 sweet spot：**NT$590-1490**
- 一次性買斷 sweet spot：**NT$300-900**

### 2.2 人脈手札的差異化定位

| 維度 | 競品平均 | 人脈手札 | 影響 |
|---|---|---|---|
| 核心功能 | 聯絡資訊 + 互動記錄 | 命理框架 + AI 深度分析 | 可定高一點（獨特性） |
| 隱私 | 多數是雲端 | 純本機 | 可定高一點（差異化） |
| AI 整合 | 多數沒有 | 3-agent 深入識人 | 可定高一點 |
| UI 拋光度 | 多數做了 5+ 年 | 上線 1 個月 | 必須定低一點 |
| 多人協作 | 多數有 | 完全沒有 | 必須定低一點 |
| 客服 | 有團隊 | 您一人 | 必須定低一點 |

**結論**：初期定價應落在台灣 sweet spot 的中段：**月 99-149 / 年 990-1490 / 一次 600-900**。

## 3. 四種收費模型（詳細設計）

### 模型 A：Freemium 訂閱 ⭐ 推薦最終形態

**免費版**（永遠免費，作為獲客漏斗）：
- ✅ 不限人脈數
- ✅ 基本命理（八字 / 星座 / 數字易經）
- ✅ 合作分析（規則引擎）
- ❌ 深入識人：每月 **3 次**（IP rate limit）
- ❌ 雲端備份
- ❌ 月報 email
- ❌ 群組分析（matrix）

**Premium NT$99/月** 或 **NT$890/年**（年訂送 2 個月）：
- ✅ 所有免費版功能
- ✅ 深入識人：**無限次**（Haiku 4.5）+ Sonnet 升級選項
- ✅ **「具體場景式」AI 合作切入點**（家人 → 過年聚餐分工 / 客戶 → 續約條件談判 / 同事 → 跨部門搶資源 等情境化建議；免費版只給「相處之道」這類通用框架）— 5/15 加入候選清單，code 端可用 `c.role` 旗標切換 prompt 嚴格度
- ✅ 雲端 E2EE 備份（自動每天備份一份到 Cloudflare R2）
- ✅ 月報 email（您這個月新增 N 人、最常互動 Top 3、推薦 follow-up）
- ✅ 群組分析（選 2-5 人做 compatibility matrix）
- ✅ 自訂主題色（小確幸）
- ✅ 優先客服

**Lifetime NT$1990 一次性**（早期使用者限定，前 100 名）：
- ✅ Premium 永久版
- ✅ 所有未來功能（包含 Phase 3 計畫中的）
- ✅ 名字寫進 `Credits` 頁
- 目的：早期注入現金、創造死忠粉絲

**轉換率假設**：
- 個人 SaaS 業界平均 free→paid: 2-5%
- 命理 + 個人工具利基：**3-4%** 偏樂觀
- Lifetime 預估：前 100 名能賣完 / 之後轉月年費

---

### 模型 B：贊助制 / 樂捐 ⭐ Phase 2 起點

**所有功能免費**，沒有付費牆。設定頁加贊助按鈕：
- Buy me a coffee NT$150
- 月贊助 NT$99
- 一次性大額 NT$500 / NT$1500

**贊助者徽章**：在他的人脈卡顯示 ❤️、寫進 `Credits` 頁。

**收入預估**：
- 贊助率約 **0.5-2%** of active users（業界 Patreon/BMC 數據）
- 平均贊助額約 NT$300（部分一次性、部分月贊助）
- 1000 users → 5-20 人贊助 → **NT$1500-6000/月**

**優點**：零摩擦、口碑好、不傷品牌
**缺點**：收入波動大、上限低（很難超過 NT$10000/月）

---

### 模型 C：一次性買斷 ⭐ Phase 2 中期試水

**免費版功能限縮**（同模型 A 免費版）+ **Pro 版 NT$590 終身**：
- 所有 Premium 功能（雲端備份除外，那是持續服務）
- 一次付款、無限期使用
- 包含 1 年免費更新（之後新功能要另外升級）

**收入特性**：
- 前 3 個月衝高（早期粉絲集中買）
- 第 4-12 個月變慢
- 不可持續，但**現金來得快**

**轉換率假設**：3-7%（買斷比訂閱容易決定）

---

### 模型 D：混合（買斷主體 + 持續服務月費）⭐ Phase 3 進階形態

**Lifetime Pro NT$1290**：解鎖所有客戶端功能（本機跑的部分）
**Cloud Service NT$59/月**：用到伺服器的功能才付（深入識人 quota、雲端備份、月報）

**邏輯**：
- 客戶端功能（命理引擎、UI 強化）→ 一次付清
- 伺服器功能（AI 配額、雲端）→ 持續付，因為您持續扛成本

**好處**：
- 使用者覺得「一次買斷有歸屬感」（不像訂閱）
- 但 AI / 雲端持續付，您不會被 free rider 燒錢
- LTV（lifetime value）比純買斷高
- 退訂只失去 cloud service，core 還在 — 退訂率低

## 4. 規模情境模擬

### 情境 ①：100 active users（Phase 1 終 / Phase 2 初）

| 模型 | 月收 NT$ | 月成本 NT$ | 月淨 NT$ |
|---|---|---|---|
| A 訂閱 (3% × NT$99) | 297 | 500 + 25 = 525 | **-228** |
| B 贊助 (1% × NT$300) | 300 | 525 | -225 |
| C 買斷 (5%, 平均到月 NT$590÷12) | 246 | 525 | -279 |

**結論**：100 users 任何模型都不賺錢。**這個階段重點是學習、不是賺錢**。靠的是您自己的興趣 + 想像力。

### 情境 ②：1000 active users（Phase 3 啟動）

| 模型 | 月收 NT$ | 月成本 NT$ | 月淨 NT$ |
|---|---|---|---|
| A 訂閱 (3% × NT$99 + 5% 年訂) | ~4,500 | 5,000 + 25 = 5,025 | **-525** |
| B 贊助 (1% × NT$300) | 3,000 | 5,025 | -2,025 |
| C 買斷（前 3 月衝 7%、後續 1%/月） | 4,100（攤平） | 5,025 | -925 |
| D 混合（5% Lifetime 攤 + 4% Cloud） | 6,800 | 5,025 | **+1,775** |

**結論**：1000 users 級別模型 D 開始有正現金流。其他模型還在虧。

### 情境 ③：5000 active users（理想規模）

| 模型 | 月收 NT$ | 月成本 NT$ | 月淨 NT$ |
|---|---|---|---|
| A 訂閱 (3% × NT$99 + 5% 年訂) | ~22,500 | 25,000 + 25 = 25,025 | **-2,525** |
| B 贊助 (1.5% × NT$300) | 22,500 | 25,025 | -2,525 |
| C 買斷 | 18,000（攤平） | 25,025 | -7,025 |
| **D 混合** | **34,000** | **25,025** | **+8,975** |
| D + 部分 user 升級 Sonnet（+NT$50） | ~42,000 | 28,000 | **+14,000** |

**結論**：5000 users 模型 D 月淨 NT$9000-14000，**可以 cover 您 10 小時/月的機會成本**。

> ⚠️ 但 5000 active users 是很大的規模。台灣命理 + 個人 CRM 利基市場可能根本到不了。要先用 Phase 1/2 驗證需求存在。

## 5. 損益平衡分析（break-even）

達到「月淨 ≥ 0」需要的 active users 規模：

| 模型 | break-even active users |
|---|---|
| A 訂閱 | ~1200 |
| B 贊助 | ~1800 |
| C 買斷 | ~1500（前期、之後會掉） |
| D 混合 | ~700 ⭐ |

達到「月淨 ≥ NT$5000 (cover 您時間)」：

| 模型 | active users |
|---|---|
| A 訂閱 | ~2700 |
| B 贊助 | ~3500 |
| C 買斷 | 不穩定，需要持續新流量 |
| D 混合 | ~1500 ⭐ |

**結論**：**模型 D 在所有情境下都最快達 break-even**。

## 6. 推薦發行順序

跟 `產品路線圖.md` 的 Phase 對齊：

```
Phase 0 (現在)        Phase 1 (1-2 月)       Phase 2 (3-6 月)         Phase 3 (6-12 月+)
─────────────         ─────────────         ─────────────           ─────────────
全免費 (現況)    →    全免費 + 設意願鈕  →    上 B 贊助 + 試水 C   →    切到 D 混合
                       (點 Premium 按鈕         買斷限量 (前 100)        (Lifetime Pro
                        看點擊率)                                        + Cloud Service)
```

**詳細**：

| 階段 | 何時 | 做什麼 | 預期收入 |
|---|---|---|---|
| **0→1 全免費** | 現在 | 不收費，加「贊助意願」按鈕（點了不收錢，只是統計點擊） | NT$0 |
| **1→2 試水** | 50+ users 後 | 啟用 B 贊助按鈕（BMC / Stripe Link） | NT$500-2000/月 |
| **2→2.5 限量買斷** | 100+ active 後 | 發 Lifetime NT$1990 ×100 名，賣完就停 | 一次 NT$50,000-200,000 |
| **2.5→3 正式 D** | 500+ active 後 | Lifetime Pro NT$1290 + Cloud Service NT$59/月 | NT$5000-15000/月 |

> Lifetime 限量是早期使用者最有效的注資手段，而且不破壞長期訂閱模型。

## 7. 單位經濟（unit economics）

模型 D 假設：

- **CAC（獲客成本）**：早期靠口碑 + Line 朋友，CAC ≈ 0。後期可能需要 NT$100/user 廣告
- **LTV（生命週期價值）**：
  - Lifetime Pro 買斷者：NT$1290 + 平均訂閱 Cloud Service 18 個月 × NT$59 = **NT$2352**
  - 純 Cloud Service 訂閱者（沒買 Lifetime）：平均 12 個月 × NT$59 = **NT$708**
  - 混合 LTV ≈ **NT$1500-2000**
- **LTV / CAC 比**：
  - 早期（零 CAC）：∞
  - 中後期（NT$100 CAC）：15-20x ✓ 非常健康（業界標準 > 3x）
- **回本期**：< 2 個月 ✓

## 8. 風險與假設

### 主要風險

| 風險 | 影響 | 緩解 |
|---|---|---|
| 命理把潛在使用者嚇跑 | 限制 TAM | Phase 2 加「純筆記模式」（無命理）做 A/B 測 |
| 台灣使用者不付費（價格敏感） | 轉換率 < 1% | Phase 2 測「贊助」轉換率，太低就停損 |
| Anthropic 漲價或停服 | 成本暴增 | 預備 OpenRouter / Workers AI fallback |
| 蘋果 iCloud Drive 政策變動 | 雲端備份壞掉 | 用 Cloudflare R2 自己掌控 |
| 您失去興趣 / 倦怠 | 產品停滯 | 限制每月投入時間，把每週日 30 分鐘列進日曆 |
| 競品出更好的 | 失去使用者 | 守好「隱私 + 命理 + 中文」三角護城河 |

### 關鍵假設（需要驗證）

1. ❓ **有 ≥ 50 個台灣使用者願意用「命理 + AI 個人 CRM」**
   - 驗證方式：Phase 1 結束時看 active 使用者 count
   - 沒過 → 砍命理、改純個人 CRM

2. ❓ **付費意願 ≥ 2%**
   - 驗證方式：Phase 2 加 Premium 按鈕（不收費，只計點擊）
   - 沒過 → 留純贊助制，不上訂閱

3. ❓ **Lifetime 早期支持者願付 NT$1990 × 100 名**
   - 驗證方式：發限量公告，看 24 小時內賣多少
   - 賣不到 30 名 → 降到 NT$1290 / 不做 Lifetime

4. ❓ **Cloud Service 月費可以撐 retention > 60% @ 6 months**
   - 驗證方式：Phase 3 開始 6 個月後算
   - retention < 40% → 改成年費（綁長）

## 9. 決策觸發點（trigger）

避免在不確定時做大決策，定好觸發條件：

| 觸發 | 動作 |
|---|---|
| 50+ users + 3+ 主動詢問「會收費嗎」 | 上線 B 贊助按鈕 |
| 連續 3 個月贊助收入 > NT$3000 | 試發 Lifetime 限量買斷 |
| Lifetime 賣出 50 名 + 月使用者 ≥ 500 | 規劃模型 D 上線 |
| 連續 2 個月 Cloud Service 訂閱 retention < 30% | 重評定價 / 功能 |
| 月成本 > 月收入 連續 3 個月 | 強制檢討（要嘛降成本、要嘛提價、要嘛收手） |
| 月淨利 > NT$30,000 連續 6 個月 | 評估全職投入 / 招人 |

## 10. 我的建議路徑（總結）

**現在 → 兩個月**：什麼都不收費。專心做 Phase 1 P0（回報按鈕、Analytics、PWA）。**在設定頁加「未來打算收費，您的意見」按鈕**，純收意見。

**第 3 月**：放上 Buy-Me-a-Coffee 連結。觀察自然贊助率。

**第 5-6 月**：如果 active 使用者 ≥ 100 + 贊助率 ≥ 1%，**發 Lifetime NT$1990 × 100 名限量**。賣完就停。這波是您**最重要的早期現金 + 死忠粉絲注入**。

**第 9-12 月**：正式上模型 D。Lifetime Pro NT$1290（之後常駐版）+ Cloud Service NT$59/月。

**第 18 月 +**：評估是否：
- 維持兼職（月淨 NT$5K-15K 就 OK）
- 全職投入（需要月淨 ≥ NT$40K 連續半年）
- 出場（找買家、轉開源、賣 IP）

---
