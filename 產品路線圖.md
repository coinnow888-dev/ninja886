---

## 產品定位

**一句話**：結合東西方命理當「快速判斷人際適配度」框架的個人 CRM。

**不是**：
- ❌ 預測未來的算命網站
- ❌ 取代 Notion / Salesforce 的全功能 CRM
- ❌ 多人協作工具

**是**：
- ✅ 個人筆記、命理當作「另一層 framework」幫助決策
- ✅ 資料留本機、隱私第一
- ✅ 行動裝置優先、桌機加分
- ✅ 單檔 HTML、無外部依賴、極低維護成本

---

## 現況 (Phase 0 — 已完成)

**目標**：把核心引擎做扎實，先自己用得順手。

**已落地**（截至 2026-05-14）：
- 八字 + 數字易經 + 星座三套命理引擎
- 東西方雙主題切換 + 命理結果自動轉換
- IndexedDB 本地儲存 + JSON 匯出/匯入
- 名片 OCR + 語音筆記
- 桌機 master-detail 佈局
- Cloudflare Workers 自動部署 + GitHub 版控
- 3-agent「深入識人」AI 分析（命理分析師 → 合作策略師 → 對話顧問）
- Cloudflare Worker proxy（使用者免 API Key，每 IP/日 3 次免費）
- iOS Safari IndexedDB 自動重連 bug 修正

**已知資料**：0 個外部使用者（產品剛公開）。

## Phase 1 — 朋友圈試用（目標 1-2 個月內）

**北極星指標**：5-30 個朋友親自用過、至少 5 個留下回饋

### 必做（這個月）

| 優先 | 項目 | 為什麼 |
|---|---|---|
| 🔥 P0 | **App 內加「📝 回報問題」按鈕** | 沒有回饋管道，您看不到痛點。MVP 是 mailto: 或 Line 連結 |
| 🔥 P0 | **Cloudflare Web Analytics 啟用** | 看每天多少人開、停留多久。已寫在持續優化指南，做就對了 |
| 🔥 P0 | **PWA 化（加到主畫面變 App）** | 手機優先的產品，必須能像 App 一樣呼叫 |
| ⚠️ P1 | **首次使用導覽** | onboarding 完選東西方後，給 30 秒新手提示卡（如何加人脈、深入識人怎麼用） |
| ⚠️ P1 | **空狀態文案優化** | 「尚未記錄任何人脈」改成「點右下角 + 加第一個人，3 個欄位就能跑命理」 |

### 也許（看回饋再決定）

- 「分享名片」連結（用 query string 帶基本資料給朋友填）
- 重要日提醒（生日、上次見面 30 天以上）
- 名片照片附加到人脈卡
- 標籤系統（除了五行/元素，自訂 tag）

### 退出條件

達到以下任一 → 進入 Phase 2：
- 至少 10 個非親友主動加入使用
- 一週收到 ≥ 3 條使用者抱怨（這是好事，代表有人在乎）
- 自己每週主動打開 ≥ 5 次

達到以下 → **不要進下個階段、回頭打磨**：
- 一週 5 個朋友 < 2 次打開
- 自己已經 2 週沒開
- 沒人寫過任何回饋

## Phase 2 — 公開試水溫（目標 3-6 個月內）

**前提**：Phase 1 退出條件通過。

**北極星指標**：50-200 個 active 使用者（一週開過 ≥ 1 次）

### 必做

| 優先 | 項目 | 為什麼 |
|---|---|---|
| 🔥 P0 | **OCR 也走 Worker proxy（免 API Key）** | 跟深入識人對齊，一致性。新使用者不用懂 API Key 是什麼 |
| 🔥 P0 | **「複製深度分析摘要」按鈕** | 使用者想分享給朋友看「我認識的你」，目前只能截圖。複製成 markdown / 圖片 |
| ⚠️ P1 | **首頁 SEO + 落地頁簡介** | 給沒登入過的人看到「這是什麼」。可在 `/about` 加個簡單介紹頁 |
| ⚠️ P1 | **資料雲端備份（選用，匯出到 iCloud Drive / Google Drive）** | 不做雲端同步，但讓使用者一鍵備份 JSON 到雲。低成本、解 80% 換手機痛點 |
| ⚠️ P1 | **桌機版鍵盤快捷鍵** | `n` 新增、`/` 搜尋、`Esc` 關 sheet。重度使用者愛 |

### 也許

- 群組分析：選 2-5 個人脈做合作 matrix（誰跟誰最合、最衝）
- 月報：每個月寄 email「您這個月新增 N 人，最常互動 Top 3 是 ...」
- 主題擴充：除東西方，加紫微斗數、人類圖、MBTI 之一

### 商業模式假設（這個 Phase 開始驗證）

| 假設 | 驗證方式 |
|---|---|
| **有人願意付費** | 加「Premium」按鈕（暫不串金流）→ 看點擊率。> 5% 點擊 = 真有人想付 |
| **付費功能誘因夠** | 詢問點擊 Premium 的人「您願意付多少看什麼？」 |
| **病毒擴散會發生** | Web Analytics 看「邀請朋友」按鈕的 referrer 帶來的流量比例 |

### 退出條件

達到以下任一 → 進入 Phase 3（規模化）：
- 50+ active users 持續 4 週
- ≥ 10 人說「願意付費」（不需真付，口頭/問卷）
- 自然搜尋進來的流量 ≥ 直接連結流量

達到以下 → **退回 Phase 1 打磨或砍掉某些功能**：
- 註冊後 7 天 retention < 20%（10 人試用，2 週後 < 2 人還在用）
- 自己也覺得做完一輪後沒新東西想做

## Phase 3 — 規模化（目標 6-12 個月，僅在 Phase 2 訊號達成時啟動）

**前提**：Phase 2 退出條件通過。

**北極星指標**：500-2000 active users，付費轉換率 ≥ 3%

### 真正需要做架構升級的時候到了

| 信號 | 對應動作 |
|---|---|
| 「換手機資料就沒了」抱怨 ≥ 5 次 | 加後端做雲端同步（Cloudflare D1 / Supabase） |
| 一週 ≥ 10 個回饋 | 上 Linear / Notion 管 backlog，停用 `待辦清單.md` |
| 一週要部署 ≥ 3 次 | 上 staging 環境 + 自動測試（vitest + playwright） |
| 有人說「值得付費」 ≥ 10 次 | 串 Stripe / Lemon Squeezy 收費 |
| 出現第二個使用情境（B2B HR、業務團隊） | 評估開 B2B 版，跟個人版分開 |

### 商業模式選項

詳細評估、定價設計、規模情境、損益平衡、決策觸發點請見：
**[商業評估.md](商業評估.md)**

簡述：四種模型（A 訂閱 / B 贊助 / C 買斷 / D 混合）。
**推薦路徑**：Phase 1 全免費 → Phase 2 上 B 贊助 + 試水 C 限量買斷 → Phase 3 切到 D 混合（Lifetime Pro NT$1290 + Cloud Service NT$59/月）。
模型 D 在所有規模情境下都最快達到 break-even（~700 active users）。

## 關鍵不確定性（隨時要回頭重評）

1. **命理會不會把潛在使用者嚇跑？**
   - 信命理的人會用、不信的人會直接關掉
   - 西方使用者要的可能不是命理而是 MBTI / Big Five
   - 解法：保留切換主題，可能要加「純筆記模式」（無命理）

2. **單一使用者（您）vs 公開市場品味差異**
   - 您自己想要的功能不等於別人想要的
   - 強制每週看 Web Analytics + 回饋避免自嗨

3. **AI 成本可持續性**
   - 100 個 active users 每天用 3 次 = $6/day = NT$5400/月
   - 達到 Phase 3 規模前必須驗證「會有人付費」否則燒不下去
   - 不能等到沒錢才開始想收費

4. **隱私 vs 雲端同步矛盾**
   - 產品定位是「資料留本機」
   - 但使用者抱怨「換手機就沒了」會是真痛點
   - 解法：E2EE 雲端同步（鑰匙在使用者手機），不是直接送雲

5. **Cloudflare workers.dev 網址不專業**
   - `ninja886.coinnow888.workers.dev` 不適合對外推廣
   - Phase 2 前應該買個 domain（NT$300/年），例如 `renmai.app` / `manmai.tw`

## 不會做的事（明確劃線）

避免被功能蔓延：

- ❌ 多人協作 / 共享人脈
- ❌ AI 自動找人脈關係（推測 A 跟 B 認識）
- ❌ 整合 LinkedIn / Facebook
- ❌ 行事曆同步
- ❌ 名片自動印製
- ❌ 即時通訊
- ❌ 商業情報（誰跳槽到哪家）

這些都是合理的擴張方向，但**每一個都會把產品變成不同的東西**。如果以後想做，重開新專案。
